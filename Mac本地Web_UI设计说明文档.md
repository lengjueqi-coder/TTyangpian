


# Mac 本地 Web UI 设计说明文档

## 1. 项目定位

本界面用于本地运行在 Mac 上的 Web 端工具，核心目标是：

- 管理默认词库
- 快速选择人物生成相关配置
- 支持生成预览
- 支持将当前配置保存为预设
- 支持通过“图片 + 名称”识别预设，便于后续复用

整体产品气质参考：

- 简洁
- 干净
- 轻量
- 白底
- 弱分割
- 圆角
- 有呼吸感
- 视觉上参考小红书发布页的清爽感
- 但交互逻辑必须是桌面端 Mac Web 工具界面，不是手机端页面

---

## 2. 设计目标

### 2.1 功能目标

系统应支持以下核心能力：

- 配置默认词库的大类与子项
- 选择每个大类当前生效的内容
- 编辑词库结构
- 实时用于生成区域
- 将当前整套配置保存为预设
- 预设支持封面图、名称、配置复用

### 2.2 体验目标

- 桌面端操作效率高
- 不要做成厚重的后台管理界面
- 不要做成移动端单列发布页
- 要像轻量化创作工具
- 用户一眼能分清：
  - 左边是配置
  - 右边是生成和预设
- 用户不需要记忆复杂名字，可通过图片识别预设

---

## 3. 整体页面结构

页面采用 **左右双栏布局**。

### 左侧：配置区域
用于展示和编辑默认词库分类及其子项。

### 右侧：生成与预设区域
用于展示生成结果、执行生成、保存预设、管理预设。

---

## 4. 页面布局说明

建议整体宽度适配桌面端，推荐最小设计宽度：

- 1440px 作为标准设计稿宽度
- 最小兼容宽度建议不低于 1280px

推荐布局比例：

- 左侧配置区域：$55\%$
- 右侧生成与预设区域：$45\%$

页面大结构如下：

```text
┌────────────────────────────────────────────────────────────────────────────────────────────────────┐
│ 顶部栏：页面标题 + 保存 + 取消                                                                     │
├────────────────────────────────────────────────────────────────────────────────────────────────────┤
│ 左侧：配置区域                                              │ 右侧：生成预览 + 我的预设            │
└────────────────────────────────────────────────────────────────────────────────────────────────────┘
5. 左侧配置区域设计说明
左侧配置区域为本系统的核心编辑区，采用“可折叠的大类列表”设计。

每个大类具备以下能力：

展开 / 收起
显示当前选中摘要
重命名大类
删除大类
展开后可管理子项
底部支持新增子项
配置区域底部需支持：

新增大类
6. 大类结构设计
每一个大类包含：

展开箭头
大类名称
当前已选摘要
编辑图标
删除图标
推荐结构：

text


▾ 年龄段                                 12-18岁   ✎   ×
未展开时：

text


▸ 年龄段                                 12-18岁   ✎   ×
其中：

▾：已展开
▸：已收起
✎：编辑 / 重命名当前大类
×：删除当前大类
7. 子项结构设计
展开大类后，展示其所有子项。

单选型子项
适用于：

性别
年龄段
国家 / 地区
气质风格
妆容风格
前缀
后缀
子项结构示例：

text


● 12-18岁                                      ✎   ×
○ 18-22岁                                      ✎   ×
○ 23-28岁                                      ✎   ×
其中：

●：当前选中
○：未选中
✎：编辑 / 重命名子项
×：删除子项
选择某个子项后，应立即：

更新大类摘要
自动收起当前大类
将结果同步到右侧生成逻辑中
多选型子项
适用于：

瑕疵细节
表情（如果后续定义为可多选）
子项结构示例：

text


☑ 雀斑                                         ✎   ×
☑ 痘印                                         ✎   ×
□ 黑眼圈                                       ✎   ×
其中：

☑：已选中
□：未选中
多选项建议在展开区域底部提供确认逻辑，或者在点击时即时生效，具体可根据实现方式决定。若即时生效，则摘要实时更新。

8. 添加逻辑
8.1 每个大类内部
展开后，最后一行必须有：

text


+ 添加
作用：

在当前大类下新增一个子项
例如在“年龄段”下新增：

47-55岁
儿童
银发老人
8.2 整个配置区域底部
必须有：

text


+ 添加大类
作用：

新增一个新的一级分类
例如可新增：

发型风格
服装风格
光线氛围
镜头类型
9. 左侧配置区域交互规则
9.1 展开与收起
默认只展开一个大类
点击未展开大类时：
当前已展开项收起
新点击项展开
点击当前已展开项标题时：
当前项收起
9.2 单选行为
用户点击某个单选子项
当前子项变为选中状态
大类摘要更新
当前大类自动收起
9.3 多选行为
用户点击某个多选子项
选中状态即时变化
摘要同步更新
是否自动收起可视情况决定
建议：

单选项自动收起
多选项不自动收起，方便连续勾选
9.4 编辑行为
点击铅笔图标后，进入当前项重命名状态。

支持：

大类重命名
子项重命名
重命名方式建议：

行内编辑
或轻量弹窗编辑
优先建议行内编辑，体验更轻。

9.5 删除行为
点击叉图标后，触发删除确认。

删除规则建议：

删除子项前：提示确认
删除大类前：提示确认，避免误删整组内容
10. 右侧生成与预设区域设计说明
右侧区域分为上下两个模块：

上部：当前生成预览
展示当前配置生成的结果图

下部：我的预设
展示用户保存过的所有预设卡片

推荐结构：

text


┌──────────────────────────────┐
│ 当前生成预览                 │
│ [图片区域]                   │
│ [立即生成] [保存为预设]      │
└──────────────────────────────┘

┌──────────────────────────────┐
│ 我的预设                     │
│ [预设卡片] [预设卡片]        │
│ [预设卡片] [+ 新建预设]      │
└──────────────────────────────┘
11. 当前生成预览模块
当前生成预览区用于显示：

尚未生成时的占位图
已生成后的结果图
应包含以下按钮：

立即生成
保存为预设
重置
可选补充按钮：

使用当前结果图作为预设封面
12. 预设模块设计说明
预设是对“当前整套配置”的保存结果。

每个预设至少应包含：

预设名称
预设封面图
当前配置数据
创建时间
更新时间
13. 预设卡片设计
每张预设卡片应以图片为主，文字为辅。

推荐卡片结构：

text


┌──────────────────────┐
│      [封面图片]      │
├──────────────────────┤
│ 清冷少女感           │
│ 12-18岁 / 中国       │
│ 清冷 / 白开水妆      │
│                      │
│ [应用] [编辑] [删除] │
└──────────────────────┘
卡片中：

图片区域面积应占卡片主体
预设名称一行展示
摘要信息展示 1 到 2 行即可
底部操作按钮简洁明确

点击 保存为预设 后，应弹出保存弹窗。

弹窗中应包含：

预设名称输入框

15. 图片上传规则
预设应支持图片作为视觉识别封面。

支持两种方式：

方式 1：使用当前生成结果图
用户当前生成结果满意时，可直接作为预设封面。

方式 2：上传本地图片
用户可从本地上传 JPG / PNG / WEBP 作为预设封面。

设计要求：

上传入口清晰可见
允许拖拽上传
上传区域可用浅绿色虚线边框突出
要让用户一眼知道这里可以加图
16. 预设复用逻辑
用户可通过“我的预设”快速复用配置。

点击 应用 后，应执行：

将该预设中保存的所有配置写回左侧配置区域
左侧所有大类摘要同步更新
如有生成逻辑，可立即用于再次生成
用户可通过封面图快速识别预设，而不是只靠文字名称记忆。

这是本产品的重要体验点。

17. 预设编辑逻辑
点击预设卡片中的 编辑 后，应支持：

修改预设名称
更换封面图
更新预设配置内容
覆盖保存当前配置
删除预设
建议采用弹窗或侧边面板编辑。

18. 视觉风格说明
整体视觉风格需符合以下方向：

偏内容工具，而不是后台系统
偏创作工具，而不是管理平台
轻量、干净、克制
留白充足
边框弱化
强调层次，不强调强视觉冲击
颜色建议
主背景：暖白色
卡片背景：纯白
分割线：极浅灰
主文字：深灰黑
次文字：中灰
标签底色：浅灰
上传区强调色：浅绿色
主操作按钮：黑色或柔和品牌红
圆角建议
卡片圆角偏大
按钮圆角中等
输入框圆角柔和
阴影建议
极轻阴影
不要厚重悬浮感
19. 图标使用规则
统一图标语义如下：

▾ / ▸：展开 / 收起
✎：编辑 / 重命名
×：删除
+：新增
☑ / □ / ● / ○：选择状态
不建议同时混用“叉”和“垃圾桶”表示同一个删除动作，避免语义重复。

+ 添加大类

不建议写“提交配置”

24. 核心交互总结
本界面的核心逻辑为：

左边
选
改
增
删
折叠管理词库
